# mdg — Markdown Grep

Grep-like search over markdown files, powered by FTS5 + vector search.

Uses SQLite with FTS5 for full-text search, sqlite-vec for vector similarity,
and node-llama-cpp for local embedding generation (GGUF models).

## Install

```bash
bun install
```

To install the latest release binary into `~/.local/bin`:

```bash
curl -fsSL https://raw.githubusercontent.com/chenhunghan/mdg/main/install.sh | bash
```

Re-running the script is safe; it updates `mdg` to the latest GitHub release if needed.

On Windows, use PowerShell:

```powershell
iwr https://raw.githubusercontent.com/chenhunghan/mdg/main/install.ps1 | iex
```

Re-running the PowerShell script is also safe; it updates `mdg.exe` to the latest GitHub release if needed.

## Usage

### Text search (grep-compatible)

```bash
# Recursive search with line numbers
mdg grep -rn "pattern" [path...]

# Case-insensitive, list matching files only
mdg grep -rli "pattern"

# Count matches per file
mdg grep -rc "pattern" docs/

# All standard grep flags work (-v, -w, -x, -A, -B, -C, --include, etc.)
mdg grep -rn -A 2 -B 1 "function" .
```

### Semantic search (vector similarity)

```bash
# Search by meaning, not exact text
mdg grep --semantic "how to configure the API"
mdg grep -s "error handling patterns" docs/

# Combine with grep flags for output formatting
mdg grep --semantic -nl "authentication flow"
```

### Hybrid search (RRF fusion)

```bash
# Combines FTS5 + vector search using Reciprocal Rank Fusion
mdg grep --hybrid "search query" [path...]
```

### Indexing

```bash
# Build/update the search index (FTS + embeddings)
mdg index

# FTS-only index (fast, no model download needed)
mdg index --no-embeddings

# Force re-index everything
mdg index --force

# Check index status
mdg status
```

## Build

```bash
# Compile to a standalone binary
bun run build

# The binary can be moved anywhere; it looks for vec0.dylib in ~/.mdg/lib/
```

## Semantic search setup

The compiled binary handles grep and FTS search standalone (60MB binary).
Semantic/hybrid search requires node-llama-cpp, which is auto-installed on
first use:

```bash
# Pre-install semantic search dependencies (~52MB)
mdg setup

# Or just use --semantic and it auto-installs on first run
mdg grep --semantic "how to configure the API"
```

The sidecar installs node-llama-cpp to `~/.mdg/node_modules/` and creates
an IPC-based embed server (`~/.mdg/embed-server.js`). The compiled binary
spawns this server as a subprocess when semantic search is needed.

The embedding model (~328MB) is downloaded on first `mdg index` or
`mdg grep --semantic` to `~/.mdg/models/`.

## How it works

- **FTS acceleration**: On `mdg grep`, if an index exists, FTS5 narrows which
  files to search (coarse filter), then just-bash's grep produces exact output.
- **Lazy indexing**: First `mdg grep` triggers a background FTS-only index if
  no index exists. Embeddings are generated separately via `mdg index`.
- **Semantic search**: Embeds the query with a local GGUF model, then finds
  nearest chunks via sqlite-vec KNN search.
- **Hybrid search**: Runs FTS and vector search in parallel, merges results
  using Reciprocal Rank Fusion (RRF) for better ranking.
- **Single DB**: All data (files, chunks, FTS index, vectors) lives in
  `~/.mdg/mdg.db`.

## Configuration

| Environment variable | Default | Description |
|---|---|---|
| `MDG_EMBED_MODEL` | `embeddinggemma-300M-Q8_0` | GGUF model URI for embeddings |

Set `MDG_EMBED_MODEL` to a Hugging Face GGUF URI to use a different model:

```bash
export MDG_EMBED_MODEL="hf:Qwen/Qwen3-Embedding-0.6B-GGUF/Qwen3-Embedding-0.6B-Q8_0.gguf"
```

## Architecture

```
src/
  cli/index.ts        CLI entry point (manual arg parsing)
  db/index.ts         SQLite singleton, macOS SQLite fix, vec table mgmt
  db/schema.ts        SQL schema (files, chunks, chunks_fts, chunks_vec, meta)
  indexer/index.ts    Indexing pipeline (scan -> hash -> chunk -> embed -> store)
  indexer/scanner.ts  Recursive .md file discovery + SHA-256 hashing
  indexer/chunker.ts  Heading-based markdown chunking with overlap
  embedder/index.ts   Embedding orchestrator (in-process or IPC)
  embedder/sidecar.ts Auto-install + IPC embed server management
  search/index.ts     FTS5, vector, and hybrid (RRF) search
  search/grep.ts      Grep execution via just-bash with FTS acceleration
  fs/mdgfs.ts         IFileSystem implementation (read-only, hybrid disk+SQLite)

~/.mdg/
  mdg.db              SQLite database (FTS5 + sqlite-vec)
  lib/vec0.dylib      sqlite-vec native extension
  models/             GGUF embedding models (downloaded on first use)
  embed-server.js     IPC embed server (auto-generated by mdg setup)
  node_modules/       node-llama-cpp sidecar (auto-installed)
```

## Requirements

- [Bun](https://bun.sh) v1.3+ (required for build and semantic search sidecar)
- macOS: Homebrew SQLite (`brew install sqlite`) for extension loading
- For semantic/hybrid search: ~52MB for node-llama-cpp sidecar + ~328MB for
  the default embedding model (both auto-downloaded on first use)
